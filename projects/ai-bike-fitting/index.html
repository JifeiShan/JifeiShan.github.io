Prototype / AI-assisted bike fitting

AI Bike Fitting Report Demo

这是一个半自动骑行姿态观察与 fitting report 实验：把侧面骑行视频、车辆参数和主观反馈转成一页可复核的中文报告。

当前状态：本地 demo 已在 private repo 跑通；公开网页目前只展示项目说明，不开放视频上传。真实上传分析需要后端/本地服务，不能只靠 GitHub Pages 完成。

它想解决什么

很多 bike fitting 讨论最后都会变成零散建议：坐垫高一点、把立短一点、膝盖轨迹看看。这个项目希望先做一个轻量流程，把观察变得结构化：视频负责提供姿态证据，规则引擎负责给出可解释判断，报告负责把结果翻译成用户和 fitter 都能讨论的语言。

当前 demo 怎么工作

  1. 输入：10–20 秒固定侧面骑行视频 + 身高、inseam、车辆参数、痛点和骑行目标。
  2. 视觉分析：本地 OpenCV 读帧，MediaPipe PoseLandmarker 识别肩、肘、髋、膝、踝等关键点。
  3. 指标计算：Python 计算下死点膝角、上死点髋角、躯干角、肘角、骨盆晃动近似值和识别成功率。
  4. 报告生成：规则引擎把指标转成问题假设、证据、建议和复拍计划。

示例输出会包含什么

数据质量视频角度、稳定性、关键点识别率
视觉指标膝角、髋角、躯干角、肘角、骨盆晃动
问题假设坐垫可能偏高、前伸可能偏长、膝盖轨迹需复查等
下一步只改一个变量，复拍同角度视频，对比前后变化

需要什么素材

如果要做第一份真实报告，请提供一段 10–20 秒侧面骑行台视频。手机固定在自行车正侧面，画面里尽量同时看到肩、肘、髋、膝、踝、脚踏、车把和坐垫。最好再补充身高、inseam、曲柄长度、坐垫高度、把立长度、主要不适点和骑行目标。

边界

  • 这不是医疗诊断。
  • 这不是专业线下 bike fitting 的替代品。
  • 2D markerless 视频分析会受拍摄角度、衣服遮挡、光线和踏频影响。
  • AI report 只能作为复测和沟通起点，最终应由真人 fitter / 教练 / 医生在合适场景下复核。
  • 真实用户视频和身体数据默认不公开，不提交到 public repo。

当前可见成果

private repo 中已经有本地上传页面、姿态分析脚本、公开视频测试脚本、报告生成脚本和视频采集模板。公开站点暂时只展示这个说明页；等有 1–2 个真实样例验证报告质量后，再决定是否做真正在线上传版本。
