普通骑友训练数据复盘

我想先免费帮 3 个普通骑友看最近 2–4 周训练数据：你提供当前 FTP、训练截图、每周可训练时间、目标和痛点，我回一页训练结构复盘、疲劳风险和下一周建议。

当前状态：第一批免费样例招募中。这个服务不是职业教练处方，也不是医疗建议；它是基于训练数据的结构化复盘，用来帮助普通骑友看清楚自己为什么进步、为什么卡住、下一周应该先改什么。

适合谁

  • 有功率计、智能骑行台、码表或训练软件记录的骑友。
  • 想提升 FTP，但不知道训练结构是否合理。
  • 总觉得练很多但 FTP 卡住，或者关键间歇总做不完。
  • 想知道 Z2、甜区、阈值、VO2max 和恢复日该怎么搭配。
  • 愿意把训练截图匿名分享出来做一次复盘。

你需要提供什么

  1. 当前基线：当前 FTP，或最近一次 FTP / CP20 / ramp test 结果。
  2. 最近训练：最近 2–4 周 Strava、Garmin、Intervals.icu、TrainingPeaks、Zwift 等截图或摘要。
  3. 时间约束：每周大概能训练几小时，工作/睡眠压力如何。
  4. 目标：例如 12 周 FTP +20W、爬坡提升、减脂、长距离耐力或恢复规律。
  5. 痛点：例如练很多但 FTP 不涨、强度课做不完、总是很累、Z2 太少、不知道怎么安排恢复。

我会回你什么

训练结构看最近几周训练频率、总量、强度课和恢复日是否清楚。
强度分布观察低强度、甜区/阈值、高强度是否比例失衡。
疲劳风险提示连续高强度、突然加量、恢复不足或 FTP 设置偏差。
下周建议给出一周内最该改的 1–3 件事，而不是一次改完整训练体系。

第一批免费样例

现在先免费做 3 个样例，用来验证这件事是否真的能帮普通骑友看懂自己的训练。你可以把数据匿名发来；默认不公开原始截图和个人数据。若之后想把案例脱敏公开，会再次确认。

最短发送格式：当前 FTP + 最近 2–4 周训练截图 + 每周可训练时间 + 目标 + 痛点。

边界

  • 这不是医疗建议。
  • 这不是职业教练处方，也不保证 FTP 提升。
  • 如果你有伤病、长期不适、心血管风险或其他健康问题，请先咨询医生或专业教练。
  • 如果数据不完整，报告会明确标注数据质量，只做训练结构观察。

后续可能形态

如果第一批样例证明有价值，它可以继续发展成单次训练复盘服务、4 周跟踪复盘、训练日志模板、AI 复盘 prompt 包，甚至更轻量的训练数据 review 工具。但第一步只验证一件事：一页结构化复盘，是否真的能帮普通骑友更清楚地训练。
